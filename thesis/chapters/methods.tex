\section{Pipeline Overview}
\label{sec:methods_pipeline}

\texttt{csttool} implements a six-stage processing pipeline that uses raw diffusion-weighted MRI data to generate hemisphere-specific CST bundles with quantitative tract metrics. Two ideas guided pipeline design: full automation without manual ROI placement, and modularity that allows each stage to be executed independently. Figure~\ref{fig:pipeline_overview} illustrates the processing stages and their data flow.

The six stages are:

\begin{enumerate}
	
	\item \hyperref[sec:methods_dataimport]{\textbf{Data Import}} — Accepts DICOM or NIfTI input, performs series selection and DICOM-to-NIfTI conversion where necessary, extracts acquisition metadata from BIDS JSON sidecars, and validates gradient table integrity.
	
	\item \hyperref[sec:methods_preprocessing]{\textbf{Preprocessing}} — Applies noise reduction to the diffusion-weighted volumes, generates a brain mask via Otsu thresholding on the $b = 0$ images, and optionally applies Gibbs unringing and motion correction.
	
	\item \hyperref[sec:methods_diffusiontensorfitting]{\textbf{Diffusion Tensor Fitting}} — Fits the diffusion tensor model to the preprocessed data and derives scalar maps including fractional anisotropy (FA), mean diffusivity (MD), radial diffusivity (RD), and axial diffusivity (AD).
	
	\item \hyperref[sec:methods_wholebraintractography]{\textbf{Whole Brain Tractography}} — Estimates a direction field using the Constant Solid Angle (CSA) ODF model, then performs deterministic streamline tracking seeded from white matter voxels to generate a whole-brain tractogram.
	
	\item \hyperref[sec:methods_atlasbasedextraction]{\textbf{Atlas-Based Extraction}} — Registers the MNI152 template to the subject's native space, warps the Harvard-Oxford cortical and subcortical atlases using the resulting deformation field, defines brainstem and motor cortex ROIs from the warped labels, and filters the whole-brain tractogram to isolate bilateral CST bundles.
	
	\item \hyperref[sec:methods_metrics]{\textbf{Tract-Level Metrics}} — Computes morphological and microstructural statistics for each hemisphere, derives along-tract profiles by resampling streamlines to equidistant points, and calculates laterality indices to determine asymmetry.
	
\end{enumerate}

Each stage reads from and writes to a structured output directory, producing intermediary NIfTI volumes, tractogram files (.trk), and JSON reports that record all processing parameters and information. This architecture ensures that any stage can be executed individually or inspected independently without the need to re-run the entire pipeline. For routine use, the command \texttt{csttool run} chains all stages together sequentially, as seen Fig.~\ref{fig:pipeline_diagram}. To chain multiple runs together, \texttt{csttool batch} can be used to process multiple subject in sequence, with per-subject timeout handling and consolidated reporting.

The pipeline is implemented entirely in Python and depends on a small set of open-source libraries. Tractography, tensor fitting, and registration are provided by DIPY~\cite{garyfallidis_dipy_2014}. NIfTI handling uses nibabel~\cite{markiewicz_2026_18944560}, image resampling uses nilearn~\cite{Nilearn}, and morphological operations use SciPy~\cite{virtanen2020scipy}. Atlas data is managed through a two-tier system: the MNI152 T1 1\,mm template~\cite{fonov2011unbiased} is bundled directly with the package and is always available; the FMRIB58 FA template~\cite{smith2006tract} and the Harvard-Oxford cortical and subcortical parcellations~\cite{jenkinson2012fsl} are distributed separately under an FSL licence (non-commercial use) and must be downloaded once with \texttt{csttool fetch-data --accept-fsl-license}. Version information for all libraries and the tool itself is recorded in every output report to support reproducibility.

The following sections describe each stage in detail, beginning with data import and concluding with metric computation.

\section{Data Import}
\label{sec:methods_dataimport}

The data import stage accepts raw diffusion-weighted MRI data in two formats: DICOM and NIfTI. Format detection is automatic --- the pipeline inspects the input directory and routes processing accordingly, so no format flag is required from the user.

\subsection{DICOM Input}
\label{subsec:methods_dicom}

When a DICOM directory is provided, the pipeline first catalogues all series contained within it. A recursive scan descends up to three directory levels, identifying DICOM-containing subdirectories either by the presence of \texttt{.dcm} or \texttt{.dicom} files or by reading the DICOM preamble magic number (\texttt{DICM} at byte offset 128). Each candidate directory is then analysed using pydicom~\cite{darcy_mason_2024_13824606} to extract series-level metadata: \texttt{SeriesDescription}, \texttt{ImageType} (distinguishing \texttt{ORIGINAL} from \texttt{DERIVED}), detected b-values, estimated volume count, and in-plane phase-encoding direction.

From this metadata each series is classified into one of nine types: raw multi-directional DWI (\texttt{DIFFUSION\_RAW}), trace-weighted (\texttt{DIFFUSION\_TRACE}), ADC map, FA map, other derived diffusion image (\texttt{DIFFUSION\_OTHER} --- matches diffusion keywords but carries a \texttt{DERIVED} or \texttt{SECONDARY} image-type flag), T1/T2 structural, localizer, or unknown. A numerical suitability score is then computed. Raw, non-derived DWI series receive a base score of 50; the \texttt{ORIGINAL} image-type flag adds 10 points; having at least two distinct b-values adds a further 15 points; and anterior-to-posterior (AP) phase encoding adds 10.\footnote{Additional scoring adjustments are applied: an extra 5 points if three or more distinct b-values are present; 5 points if an explicit $b = 0$ is detected; 20 or 10 points for series with $\geq 100$ or $\geq 50$ files respectively; $-10$ for fewer than 20 files; and $-10$ for posterior-to-anterior (PA) phase encoding.} Conversely, a \texttt{DERIVED} or \texttt{SECONDARY} image type subtracts 30, and non-diffusion series are penalised by 100, making them ineligible. Any series with a net positive score is considered suitable for tractography, and the highest-scoring series is selected automatically. The user may override automatic selection by specifying a series index or Series Instance UID on the command line.

The selected series is converted to compressed NIfTI format using \texttt{dicom2nifti}~\cite{dicom2nifti}, which handles vendor-specific encoding including Siemens mosaic multi-band acquisitions and multi-frame Enhanced DICOM objects, and extracts the diffusion gradient table (\texttt{.bval}/\texttt{.bvec}) directly from the DICOM headers. When \texttt{dicom2nifti} fails, a fallback conversion via \texttt{dcm2niix}~\cite{li2016first} is attempted if that tool is available on the system path.

\subsection{NIfTI Input}
\label{subsec:methods_nifti}

When a pre-converted NIfTI directory is provided, the stage loads the compressed volume (\texttt{.nii.gz}) and the accompanying gradient files directly. Both the \texttt{.bval}/\texttt{.bvec} and the \texttt{.bvals}/\texttt{.bvecs} filename variants are accepted, accommodating data exported by different software packages.

\subsection{Gradient Table and BIDS Metadata}
\label{subsec:methods_gradienttable}

Regardless of input format, the b-values and b-vectors are used to construct a DIPY \texttt{GradientTable} object~\cite{garyfallidis_dipy_2014}, which encodes the full diffusion-encoding scheme and is passed to subsequent pipeline stages. Volumes with $b \leq 50\;\text{s/mm}^2$ are treated as $b = 0$ reference images, following the default threshold used by the DIPY library~\cite{garyfallidis_dipy_2014}.

BIDS is shorthand for brain-imaging data structure and it attempts to provide a set of guidelines for file and metaddata naming and organization, statistical modeling of neuroimaging data and applications created to be used with said data~\cite{gorgolewski2016brain, karakuzu2022qmri}. If a BIDS-compatible JSON sidecar is present alongside the NIfTI file, the pipeline reads it to extract acquisition metadata. Echo time is stored in seconds in the BIDS convention and is converted to milliseconds internally. Field strength is also extracted where present. These values are recorded in the per-subject JSON import report alongside the library versions used, providing a complete record of acquisition context for downstream reporting.

\subsection{Denoising}
\label{subsec:methods_denoising}

The first step of the preprocessing pipeline is to attempt and remove noise from given MRI images. \texttt{csttool} offers two denoising techniques which are available in DIPY: non-local means (NLMEANS)~\cite{coupe2008nlmeans, coupe2012nlmeans} and Patch2Self~\cite{fadnavis2011patch2self, fadnavis2024patch2self2}.

\paragraph{NLMEANS} A straightforward way to denoise an image is to take the mean value of a group of pixels surrounding the target pixel. This is local means filtering. A non-local means filter looks beyond the immediate neighborhood of the target, taking a mean of all the pixels in the image and weighting them by a similarity function.

Given an image with an area $\Omega$ and two points $p$ and $q$, the algorithm for NLMEANS is~\cite{buades2011non}:

\begin{equation}
	\mathrm{NL}u(p) = \frac{1}{C(p)} \int_{\Omega} f(d(B(p), B(q)))u(q)dq
\end{equation}
\label{eq:nlmeans}

where $d(B(p), B(q))$ is an Euclidean distance between image patches centered respectively at $p$ and $q$, $f$ is a decreasing function and $C(p)$ is the normalizing factor, given by:

\begin{equation}
	C(p) = \int_{\Omega} f(p,q)dq
\end{equation}
\label{eq:nlmeans_normalizingfactor}

The weighting function $f(p,q)$ is used to determine how similar the image is at point $p$ to the image at point $q$. Commonly, the Gaussian weighting function is used:

\begin{equation}
	f(p,q) = e^{-\frac{|B(q)-B(p)|^2}{h^2}}
\end{equation}
\label{eq:gaussian_weight}

where $h$ is the standard deviation, and $B(p), B(q)$ are the local mean values surrounding the image point values $p$ and $q$ respectively.

\paragraph{Patch2Self} Classical denoising algorithms such as NLMEANS need to make assumptions on the input signal, which makes them prone to bias. Instead of this, Patch2Self only uses the fact that the noise across different 3D volumes of the diffusion signal originates from random fluctuations in the acquired signal. Patch2Self removes noise by predicting each diffusion volume $V_j$ from other volumes $V$, assuming noise is random and uncorrelated across them.

\begin{figure}[htbp]
	\centering
	\includegraphics[width=0.8\linewidth]{figs/Patch2Self_Framework.png}
	\caption{Working principle of the Patch2Self algorithm~\cite{fadnavis2011patch2self}}
	\label{fig:patch2self}
\end{figure}

As shown in Fig.~\ref{fig:patch2self}, Patch2Self operates in two stages. In the first stage (A), one diffusion volume $V_j$ is held out as the target. Patches from the remaining $(n-1)$ volumes $V_{-j}$ are used to predict the center voxel of the corresponding patch in $V_j$. The target volume itself is not used as input, which prevents the model from learning noise.

In the second stage (B), the trained regressor is applied to the same input volumes $V_{-j}$ to obtain a denoised estimate $\hat{V}_j$ of the held-out volume. This procedure is repeated for all volumes. Since noise is uncorrelated across volumes, it cannot be predicted and is therefore suppressed, while consistent structural information is preserved.

\subsection{Gibbs Unringing}
\label{subsec:methods_gibbs}
%TODO: Check this subsection if it is clear enough.

Gibbs artifacts occur as a consequence of using Fourier transforms to reconstruct MR signals into images. Because an image has to be recreated by sampling only a finite number of frequencies, the Fourier series has to be cut short during reconstruction. The error due to this truncation is evident in high-contrast interfaces of the image. The resulting artifact usually manifests as intensity oscillations adjacent to edges of different tissue types~\cite{veraart2016gibbs}.

\begin{equation}
	I_{\mathrm{recon}} = I_{\mathrm{true}} * \mathit{sinc}
\end{equation}
\label{eq:mri_reconstruction}

\texttt{csttool} uses DIPY's Gibbs removal module. The algorithm is based on an adapted version of a sub-voxel Gibbs suppression procedure~\cite{kellner2016gibbs, neto2018advanced}. The technique takes advantage of the fact that the strength of these oscillations depends on how the underlying sinc pattern is sampled on the discrete voxel grid. If the sampling occurs near extrema of the sinc function, ringing is pronounced, whereas sampling near its zero-crossings minimizes the artifact. If sampling location is adjusted locally so that it occurs near zero-crossings, the ringing amplitude will be minimized.

First, a set of sub-voxel shifted images is generated by applying phase ramps in Fourier space:

\begin{equation}
	I_s(x)~\mathrm{for~different}~s
\end{equation}

For each voxel \(x\), the optimal shift is determined by minimizing local intensity oscillations, measured using a total variation criterion:

\begin{equation}
	r(x) = \arg\min_s \sum_n \left| I_s(x+n) - I_s(x+n-1) \right|
\end{equation}

This selects the shift that yields the least oscillatory neighborhood.

Finally, the corrected image is obtained by interpolating the optimally shifted signal back onto the original grid:

\begin{equation}
	I_{\text{unring}}(x) = I_{r(x)}\!\left(x - \frac{r(x)}{2M}\right)
\end{equation}

\subsection{Motion Correction}
\label{subsec:methods_motioncorrection}

Due to subject motion during acquisition, volumes become misaligned. The most common way to solve this problem is by performing a registration on each acquired volume to a reference $b=0$~\cite{jenkinson2001global}. The idea is to use a sequence of affine transformations in increasingly finer resolutions. DIPY's \texttt{register\_dwi\_series} function implements this idea in the \texttt{motion\_correction} wrapper.

A stable reference $b_0$ is built first. Afterwards, the motion correction pipeline is implemented, consisting of four stages: center-of-mass alignment, translation, rigid transformation and affine transformation.

First, center-of-mass alignment provides a coarse initialization by aligning the centroids $c_{\text{ref}}$ and $c_{\text{mov}}$ of the reference and moving volumes:

\begin{equation}
	t_{\text{COM}} = c_{\text{ref}} - c_{\text{mov}}
\end{equation}
\label{eq:motioncorrection_com}

Next, a translation-only model is optimized, where $x \in \mathbb{R}^3$ denotes a voxel coordinate and $t \in \mathbb{R}^3$ a translation vector:

\begin{equation}
	T(x) = x + t
\end{equation}
\label{eq:motioncorrection_translation}

This is followed by a rigid transformation, introducing a rotation matrix $R \in SO(3)$:

\begin{equation}
	T(x) = R x + t, \quad R \in SO(3)
\end{equation}
\label{eq:motioncorrection_rigid}

Finally, a full affine transformation is estimated with $A \in \mathbb{R}^{3 \times 3}$ capturing scaling and shear:

\begin{equation}
	T(x) = A x + t
\end{equation}
\label{eq:motioncorrection_affine}

At each stage, the transformation $T$ mapping the moving image $I_{\text{mov}}$ to the reference image $I_{\text{ref}}$ is obtained by minimizing a similarity cost function $C(\cdot)$:

\begin{equation}
	T^* = \arg\min_T C(I_{\text{ref}}, I_{\text{mov}} \circ T)
\end{equation}
\label{eq:motioncorrection_cost}

Here, $\circ$ denotes composition, i.e. resampling of $I_{\text{mov}}$ under the transformation $T$. The optimization is performed in a multi-resolution scheme, starting on coarse images and refining on finer scales to improve convergence.

\subsection{Reslicing}
\label{subsec:methods_reslicing}

At times, MR images need to be resliced in different resolutions. This is especially needed if datasets come with non-isotropic voxel sizes. \texttt{csttool} uses DIPY's \texttt{reslice} method. First, the function computes the scaling factor

\begin{equation}
	R = \frac{\text{new\_zooms}}{\text{zooms}}
\end{equation}

which defines the relative change in voxel size along each spatial dimension. If no target shape is provided, the new image dimensions are determined as

\begin{equation}
	\text{new\_shape} = \operatorname{round}\left(\frac{\text{zooms}}{\text{new\_zooms}} \cdot \text{shape}\right)
\end{equation}

so that the physical field of view is preserved while the voxel size changes. The image is then resampled onto the new grid using \texttt{affine\_transform} with \(R\) as the transformation matrix. For 4D datasets, this operation is performed independently for each volume.

Finally, the affine transformation is updated to remain consistent with the new voxel grid. A diagonal scaling matrix is constructed from \(R\),

\begin{equation}
	R_x =
	\begin{bmatrix}
		R_1 & 0 & 0 & 0 \\
		0 & R_2 & 0 & 0 \\
		0 & 0 & R_3 & 0 \\
		0 & 0 & 0 & 1
	\end{bmatrix}
\end{equation}

and applied to the original affine as

\begin{equation}
	\text{affine}_{\text{new}} = \text{affine} \cdot R_x
\end{equation}

resulting in a resampled image with updated voxel spacing and consistent spatial coordinates.

\subsection{Brain Masking}
\label{subsec:methods_brainmasking}

For skull stripping, \texttt{csttool} uses DIPY's \texttt{median\_otsu} algorithm, which combines median filtering with Otsu thresholding to generate a binary brain mask.

Given a diffusion dataset $V \in \mathbb{R}^{X \times Y \times Z \times N}$, a reference image is first computed by averaging selected b$_0$ volumes with indices $\mathcal{I}$:

\begin{equation}
	I(x) = \frac{1}{|\mathcal{I}|} \sum_{i \in \mathcal{I}} V(x,i)
	\label{eq:median_otsu_b0}
\end{equation}

To reduce noise, a median filter is applied iteratively. For a neighborhood $\mathcal{N}_r(x)$ with radius $r$, one filtering step is defined as:

\begin{equation}
	I^{(k+1)}(x) = \mathrm{median} \{ I^{(k)}(y) \mid y \in \mathcal{N}_r(x) \}
	\label{eq:median_otsu_median}
\end{equation}

and repeated for $K$ passes, resulting in a smoothed image $I^{(K)}$.

A global threshold $t^*$ is then computed using Otsu's method by maximizing the between-class variance:

\begin{equation}
	t^* = \arg\max_{t} \ \sigma_B^2(t)
	\label{eq:median_otsu_otsu}
\end{equation}

The binary mask $M$ is obtained as
\begin{equation}
	M(x) =
	\begin{cases}
		1, & I^{(K)}(x) > t^* \\
		0, & \text{otherwise}
	\end{cases}
	\label{eq:median_otsu_mask}
\end{equation}

Optional post-processing steps include binary dilation and removal of small holes or isolated regions. The final mask is applied to the input data:

\begin{equation}
	V_{\text{masked}}(x,i) = M(x) \cdot V(x,i)
	\label{eq:median_otsu_apply}
\end{equation}

Median filtering improves robustness to noise, while Otsu thresholding separates brain tissue from background based on the intensity histogram~\cite{yang2012improved}.

\section{Diffusion Tensor Fitting}
\label{sec:methods_diffusiontensorfitting}

%TODO: Question: FA map as redundancy if anat T1 not found for future pipeline? Unsure if implemented in code. Check. Idea is: look for T1 anat first, use that if available, otherwise default to legacy FA method.

Fitting the diffusion tensor model is handled by the \texttt{fit\_tensors} method within the \texttt{track} module. For each voxel of the preprocessed volumes, this method estimates the diffusion tensor $\mathcal{D}$. This section describes how \texttt{cstttool} accomplishes this. The theoretical foundations are explained in Section~\ref{subsec:background_diffusiontensor}. The derived scalar metrics are defined in Section~\ref{subsec:background_dti}.

\paragraph{Tensor model and fitting.}

Eq.~\ref{eq:signal} models the diffusion-weighted signal using a scalar diffusion coefficient $D$. In the case of anisotropic diffusion, this parameter is replaced by the full diffusion tensor $\mathcal{D}$, yielding the tensor signal model~\cite{basser1994mr}:

\begin{equation}
	S_i = S_0 \exp\!\bigl(-b\,\hat{\mathbf{g}}_i^\top \mathcal{D}\,\hat{\mathbf{g}}_i\bigr)
	\label{eq:tensor_signal}
\end{equation}

where $S_i$ is the signal measured along the $i$-th gradient direction with unit vector $\hat{\mathbf{g}}_i$, $S_0$ is the initial, unweighted signal and $b$ is the diffusion weighting factor (Eq.~\ref{eq:stejskal-tanner}). Since $\mathcal{D}$ is symmetric, it has six unique elements. Each gradient direction contributes exactly one equation of the form defined by Eq.~\ref{eq:tensor_signal}. If more than 6 non-colinear gradient directions are given, the system is overdetermined. As such, it can be solved via the ordinary least squares technique, which is the method applied by DIPY's \texttt{TensorModel} and used by \texttt{csttool}. Fitting is restricted to voxels within the brain mask obtained during preprocessing (Section~\ref{subsec:methods_brainmasking}).

\paragraph{Scalar map computation.} 

Eigenvalue decomposition of the fitted tensor (Eq.~\ref{eq:diffusion_matrix_form}) yields the eigenvalues $\lambda_1 \geq \lambda_2 \geq \lambda_3$, from which the four scalar maps FA, MD, AD and RD are computed as defined in Eqs.~\ref{eq:FA},~\ref{eq:MD},~\ref{eq:AD} and~\ref{eq:RD}. The implementation uses DIPY's \texttt{tenfit.fa} attribute for FA and the helper functions \texttt{mean\_diffusivity}, \texttt{radial\_diffusivity} and \texttt{axial\_diffusivity} from \texttt{dipy.reconst.dti} for MD, RD and AD respectively.

Noise can cause near-zero or negative eigenvalues to appear. This results in some voxels producing NaN or $\pm\infty$ values in derived scalar maps. These are replaced with zero, preventing numerical instability down the pipeline at the cost of underestimating diffusivity in affected voxels.

\paragraph{White matter mask generation.} 

The end goal of \texttt{csttool} is to classify tracts belonging to the CST. As such, the detected streamlines must be contained to white matter only. If a co-registered T1-weighted anatomical image is not provided, the FA map provides a suitable proxy for white matter mask generation. This makes the tool more self-contained. A binary white matter mask $W$ is defined as:

\begin{equation}
	W(x) =
	\begin{cases}
		1, & \mathrm{FA}(x) > \tau \;\wedge\; M(x) = 1 \\
		0, & \text{otherwise}
	\end{cases}
	\label{eq:wm_mask}
\end{equation}

where $\tau$ is a configurable FA threshold (default $\tau = 0.2$) and $M$ is the brain mask from Section~\ref{subsec:methods_brainmasking}. The mask is subsequently dilated by one voxel using \texttt{binary\_dilation} from SciPy to ensure that streamlines can reach the cortical grey--white matter interface, which is relevant for both seeding and termination during tractography.
